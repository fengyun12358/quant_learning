# 策略研究模板 — Strategy Research Template

> 每一次策略研究，从假设到结论，填入此模板。
> 不跳步骤，不事后补填。

---

## 1. 研究假设

> **要回答的问题**：我相信什么？这个信念可以被数据证实或证伪吗？

- [ 范例 ] "MA 和 RSI 各有优劣，在不同的市场状态下有不同的表现。"
- [ 范例 ] "Efficiency Ratio 可以区分趋势市场和震荡市场。"

**证伪条件**（提前设定，不可事后修改）：

- [ 范例 ] "如果时间切分后，训练集胜率 60% 而测试集胜率 < 50%，假设作废。"
- [ 范例 ] "如果 ER 预测的策略在测试集上跑输买入持有，ER 无效。"

---

## 2. 数据来源

> **要回答的问题**：用什么数据？为什么选这些？

- [ 范例 ] 四只 ETF（510330 沪深300 / 510500 中证500 / 588000 科创50 / 513100 纳指）
- [ 范例 ] 时间段：2024.09 - 2026.05
- [ 范例 ] 选择逻辑：覆盖宽基 + 行业 + 跨境，样本多样化，避免单一标的偏差
- [ 范例 ] 数据格式：国金证券 xls 导出 → DataLoader 标准化至 `(日期, 开盘, 最高, 最低, 收盘, 成交量)`

---

## 3. 实现方案

> **要回答的问题**：策略的信号规则是什么？如何接入回测框架？

- [ 范例 ] 信号接口标准化：`buy_signal` / `sell_signal`（布尔列）
- [ 范例 ] 引擎不依赖策略细节：`BacktestEngine.run(df)` 只读信号列
- [ 范例 ] 策略类共享 `fit(df)` 接口，换策略不换引擎

---

## 4. 回测设计

> **要回答的问题**：在什么数据上跑？跑了哪些对比？

**单段回测：**
- [  ] 模拟数据（test_etf.csv）：验证代码逻辑正确性
- [ 范例 ] 真实 ETF 数据：四只标的分别跑 MA 和 RSI

**多资产对比：**
- [ 范例 ] 同一策略在四只 ETF 上的结果对比

**时间切分：**
- [ 范例 ] 510330 沪深300：前 2/3 训练（~280天），后 1/3 测试（~140天）
- [ 范例 ] 目的：检测过拟合——训练集上赢的策略是否在测试集上继续赢

---

## 5. 风险分析

> **要回答的问题**：这个策略最怕什么？什么情况下会亏钱？

- [ 范例 ] RSI 策略最怕"一直涨或一直跌"的单边市场 — 均值回归的前提被破坏
- [ 范例 ] MA 策略最怕反复震荡 — 假突破频繁触发交易，手续费侵蚀收益
- [ 范例 ] 任何择时策略的共同风险：信号滞后于市场变化

---

## 6. 研究结论

> **要回答的问题**：证明了什么？证伪了什么？下一步做什么？

**已验证：**
- [ 范例 ] ER（效率比率）能有效区分市场状态，预测 MA vs RSI 的胜负关系
- [ 范例 ] 不同市场适合不同策略，不存在"圣杯"

**已证伪：**
- [ 范例 ] ER(50) 窗口过长，滞后 30-36 天，无法用于实时策略切换
- [ 范例 ] 时间切分后排名反转 → 单段回测的"最优参数"不可信

**失败的价值：**
- [ 范例 ] ER 切换策略虽然跑输，但 ER 本身的预测能力值得用更快的指标（如 ADX、20 日波动率）重新验证

**下一步方向：**
- [  ] 缩短 ER 窗口或换用更灵敏的趋势强度指标
- [  ] 对每只 ETF 单独优化参数，而非用统一参数
- [  ] 引入多资产组合，利用策略间的低相关性降低回撤

---

## 7. 研究方法论（不可跳过的 5 步）

```
① 获取多份数据集       ← 单一数据上的结论不可推广
② 将数据集做时间切片    ← 训练/测试分离，杜绝未来函数
③ 策略运行于不同数据集   ← 跨标的验证，排除幸存者偏差
④ 策略在不同时间切片运算 ← 最严格的过拟合检测
⑤ 对比所有结果与基准     ← 问自己：如果买入持有更好，择时有什么意义？
```
